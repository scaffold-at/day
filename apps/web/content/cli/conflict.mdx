# `scaffold-day conflict`

> list / resolve / detect scheduling conflicts

## WHAT

Inspect, resolve, or trigger detection of conflicts. Conflicts are detected automatically when `place do` / `place override` writes the day file; this command is the user-facing surface for them.

## WHEN

When `today` shows open conflicts, or after a manual edit you want to verify.

## COST

Local file I/O over `<home>/conflicts/<YYYY-MM>.json` and the day files.

## INPUT

```
list [--month <YYYY-MM>] [--status open|resolved|ignored|all] [--json]
resolve <id> --status resolved|ignored [--reason <T>] [--by <attr>] [--json]
detect <YYYY-MM-DD> [--json]
```

## RETURN

Exit 0. DAY_NOT_INITIALIZED if no policy. DAY_NOT_FOUND for unknown conflict. DAY_INVALID_INPUT on bad month / non-open conflict.

## GOTCHA

v0.1 `resolve` records the decision and clears `day.conflicts_open` but does NOT mutate the underlying placements — use `place override` for the actual move/remove. Auto-resolve lands in v0.2. Tracking SLICES.md §S24 (cmd) / §S23 (detection).
