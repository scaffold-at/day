# `scaffold-day day`

> explore day partitions (months / overview / get / range)

## WHAT

Read-only navigation over the days/ tree. Sub-commands: `months` (list partitions), `overview <YYYY-MM>` (per-day counts), `get <YYYY-MM-DD>` (full view), `range <start> <end>` (compact list).

## WHEN

When you want a quick AI-friendly summary or a one-day detail view.

## COST

Local file I/O. `get` and per-day computation in `range` recompute free slots each call.

## INPUT

```
months
overview <YYYY-MM>
get <YYYY-MM-DD> [--json] [--tz <iana>]
range <start> <end> [--json] [--tz <iana>]
```

## RETURN

Exit 0 on success. DAY_USAGE on missing args. DAY_INVALID_INPUT on bad date / month strings. `--json` returns the structured DayView (or range summary).
