# `scaffold-day telemetry`

> inspect or change opt-in heartbeat telemetry preferences

## WHAT

Read or set the telemetry preference. Pseudonymous: events are tagged with an install_id (UUID per home), never TODO content / calendar / paths. Transport is PostHog Cloud when SCAFFOLD_DAY_POSTHOG_URL + SCAFFOLD_DAY_POSTHOG_KEY are set; otherwise events queue silently.

## WHEN

Whenever you want to opt in, opt out, check status, or rotate the install_id.

## COST

Local config write only. Each captured event (state=on, transport configured) is one HTTPS POST to PostHog.

## INPUT

```
status (default) | on | off | ask | show-id | reset-id [--json] [--dry-run]
```

## RETURN

Exit 0. JSON mirrors the on-disk config plus transport state.

## GOTCHA

Default is `ask` — nothing is sent until you opt in. Even with state=on, transport is a no-op until SCAFFOLD_DAY_POSTHOG_URL + _KEY are set. Tracking SLICES.md §S45 / issue #3 §S65.
